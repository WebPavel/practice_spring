事务特性ACID
原子性：事务要么全都执行，要么全都不执行
一致性：事务执行前后，数据保持完整性
隔离性：一个事务的执行，不影响其他事务的执行
持久性：事务一旦提交，对数据库的影响是永久性的
脏读：一个事务读到另一个事务未提交的数据
不可重复读：一个事务读到另一个事务已经提交的数据（update）,导致一个事务多次查询结果不一致
虚读：一个事务读到另一个事务已经提交的数据（insert）,导致一个事务多次查询结果不一致
read_uncommitted未提交读：以上问题都可能发生
read_committed已提交读，可以避免脏读，不可重复读和虚读可能发生
repeatable_read可重复读，避免脏读和不可重复读，虚读可能发生
